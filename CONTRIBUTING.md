# Contributing to VX Launcher

Thank you for your interest in contributing to VX Launcher! We welcome help in making the launcher faster, more stable, and easier to use.

Here are ways you can contribute:
- Reporting bugs or usability issues.
- Suggesting new features.
- Fixing issues by opening a pull request.
- Improving documentation.

## Reporting Issues

If you find a bug or have a suggestion, please check the existing issues to ensure it hasn't already been reported. If not, open a new issue containing:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Details about your environment (e.g., Windows version, launcher version).
- Expected vs. actual behavior.

## Submitting Pull Requests

1. **Fork the Repository:** Create your own fork of this repository.
2. **Create a Branch:** Create a branch for your work (e.g., `git checkout -b feature/cool-new-feature` or `git checkout -b bugfix/fix-launch-error`).
3. **Commit Changes:** Keep commits focused, write clean commit messages, and preserve project code formatting.
4. **Push & Pull Request:** Push your changes to your fork and submit a Pull Request to our main branch. Describe your changes clearly in the PR description.

## Code of Conduct

Please maintain a friendly, cooperative, and constructive tone in all project communications (issues, pull requests, comments).

Thank you for contributing!
